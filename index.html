なぜMarkdown？

Web向けの文章フォーマットとして、すでに現場で使われています。

活用事例

  • GitHubのREADME
  • APIリファレンス
  • ブログ記事（Qiita / Zenn）
  • WordPressでのホームページ

エンジニアはMarkdownが得意。だから、資料作成の入口をMarkdownに寄せると速い。

Markdownを“単一ソース”にするメリット

  1. 再利用：スライド→ブログ→LPへ展開
  2. 差分管理：Gitで履歴が追える
  3. AI相性：構成・要約・整形が得意

Marpとは

Markdownからスライドを作るツール。VS Code運用が超ラク。

従来のスライド作成

  • レイアウト調整が大変（4:3→16:9など）
  • ファイル管理が面倒

Marpなら

  • 構成に集中できる
  • Git管理できる
  • 再利用しやすい
最小ルール
--- でスライド分割 / Markdownで書くだけ

“LT向け”に強い理由

  • 修正が速い（文書編集）
  • 見た目はCSSで統一
  • テンプレ化が簡単

MarkdownからLPを作る方法

結論：ChatGPTにMarkdownを渡して、HTML(CSS)へ変換を指示すればOK。

最短フロー（おすすめ）

1
Markdownを用意
構成・文章・見出し
2
ChatGPTへ貼る
「LP用HTML+CSSにして」
3
index.html / styles.css
コピペでOK
4
公開
GitHub Pages / Netlify など
ポイント：LPは「読みやすさ」と「CTA」優先。スライドの横から、LPの縦へ再構成する。
